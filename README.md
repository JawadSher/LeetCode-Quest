## 📖 Introduction

Welcome to my **LeetCode Quest** repository.

LeetCode Quest is a domain-driven challenge on LeetCode — each domain covers a specific area of computer science and is split into levels of increasing difficulty. It's not just about racking up a solve count; it's about building genuine depth across Data Structures & Algorithms, Databases, System Design, Mathematics, and more.

I built an entire [DSA repository](https://github.com/JawadSher/DSA-LeetCode-GFG-Problems-Repository) from scratch after grinding through **Love Babbar's complete DSA series** — seven-plus months of consistent effort. I also completed the [LeetCode Top Interview 150](https://github.com/JawadSher/LeetCode-Top-Interview-150-Problems) study plan. This repo is the natural next step: domain-by-domain mastery across every Quest category, solved cleanly in C++ and documented so anyone can follow along.

Every problem folder contains a `solution.cpp` and a `README.md` with the problem statement, my approach, and time/space complexity analysis.

## 🧠 Domain-First vs. Problem-First: Why It Matters

Most people grind LeetCode problem by problem — random order, or sorted by difficulty. LeetCode Quest flips that. You go deep into one domain at a time, which means:

- You stop reinventing patterns and start recognising them instantly
- Your solution speed improves because your mental model is complete before you start
- Domain mastery transfers directly to real interviews, where problems rarely announce their category

The approach I follow per domain:

1. **Survey the domain** — read the key data structure or algorithm, nothing exhaustive
2. **Solve 2–3 warm-up problems** — rebuild muscle memory before the hard levels
3. **Identify the 2–3 core patterns** in that domain and note them down
4. **Revisit the first problem 48 hours later** — this is what locks it in
